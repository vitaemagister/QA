# QA Templates

Ready-to-use templates for QA documentation in English and Ukrainian.

---

## Bug Report Templates

**File:** [`bug_report_template_en.docx`](bug_report_template_en.docx) (Ukrainian version also available)

**Includes:**
- Bug ID & Title
- Steps to reproduce
- Expected vs Actual results
- Severity & Priority
- Environment details
- Screenshots section

**Use for:** Documenting bugs, bug tracking systems

---

## Test Case Template

**File:** [`test_case_en.xlsx`](test_case_en.xlsx)

**Columns:**
- Test Case ID
- Test Case Description
- Preconditions
- Test Steps
- Expected Result
- Actual Result
- Status (Pass/Fail/Blocked)
- Priority
- Comments

**Use for:** 
- Creating test suites
- Regression testing
- Manual test execution tracking

---

## Test Plan Templates

**File:** [`test_plan_en.docx`](test_plan_en.docx) (Ukrainian version also available)

**Sections:**
- Project overview
- Testing scope
- Test strategy
- Test schedule
- Resources
- Risks & Mitigation
- Deliverables

**Use for:** Formal test planning, stakeholder communication

---

## How to Use

1. **Download** the template you need
2. **Open** in Microsoft Word or Excel (or compatible software like LibreOffice)
3. **Fill in** your project details
4. **Customize** sections as needed
5. **Save** with your project name

---

## Template Compatibility

| Template | Format | Compatible With |
|----------|--------|----------------|
| Bug Reports | `.docx` | Word, Google Docs, LibreOffice Writer |
| Test Cases | `.xlsx` | Excel, Google Sheets, LibreOffice Calc |
| Test Plans | `.docx` | Word, Google Docs, LibreOffice Writer |

---

## Tips

- **Bug Reports:** Always include clear reproduction steps and expected behavior
- **Test Cases:** Keep test steps atomic and specific
- **Test Plans:** Update regularly throughout the project lifecycle
- **Screenshots:** Add visual evidence whenever possible
- **Version Control:** Track template versions in your project documentation
